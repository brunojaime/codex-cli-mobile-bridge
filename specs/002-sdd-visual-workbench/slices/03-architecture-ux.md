# Slice 3: Architecture UX

## Goal

Make SDD artifacts useful as an architecture review surface, not just files.

## Views

- Overview: project status and SDD coverage.
- Specs: specs, plans, tasks, and acceptance criteria.
- Diagrams: grouped by diagram family.
- Feedback: pending and resolved feedback linked to SDD artifacts.
- Actions: Codex actions available for the selected artifact.

## Done When

- A user can quickly understand what exists, what is missing, and what changed.
- Diagram families are navigable by engineering nomenclature.
- The UI remains project-local unless running inside the bridge/control app.
- The UI remains useful for monolith, modular, microservice, and hybrid
  systems without forcing a project category.
